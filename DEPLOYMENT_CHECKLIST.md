# 📋 Deployment Checklist - Before Going Live

Use this checklist to ensure everything is properly configured before deploying to production.

---

## ✅ Pre-Deployment Phase

### 1. Code Quality
- [ ] No console errors in browser dev tools
- [ ] No TypeScript compilation errors (`npm run build` completes successfully)
- [ ] All Angular components compile without warnings
- [ ] No unused variables or imports

### 2. Environment Variables
- [ ] `.env` file exists locally with all required variables:
  - [ ] `MONGO_URI` (local or MongoDB Atlas)
  - [ ] `SESSION_SECRET` (32+ characters)
  - [ ] `CLIENT_ID` (from Google Cloud Console)
  - [ ] `CLIENT_SECRET` (from Google Cloud Console)
  - [ ] `NODE_ENV=development`
  - [ ] `PORT=8080`
- [ ] `.env` is in `.gitignore` (don't commit!)
- [ ] `.env.example` exists with placeholders

### 3. Local Testing
- [ ] Run `npm install` and verify no errors
- [ ] Run `npm run build` and verify build succeeds
- [ ] Run `npm run dev` and app starts on http://localhost:8080
- [ ] Landing page loads
- [ ] Influencer login button works
- [ ] Brand login button works
- [ ] Google OAuth login works
- [ ] MongoDB connection is successful

### 4. Docker Testing (Optional but Recommended)
- [ ] Docker installed on local machine
- [ ] Run `docker-compose up` successfully
- [ ] App accessible at http://localhost:8080
- [ ] MongoDB container is running
- [ ] No Docker build errors

### 5. Git Preparation
- [ ] All changes committed (`git status` shows clean)
- [ ] Repository pushed to GitHub (`git push`)
- [ ] `.gitignore` includes:
  - [ ] `node_modules/`
  - [ ] `.env`
  - [ ] `dist/`
  - [ ] `*.log`
  - [ ] `.angular/`

## 🔧 Platform Setup Phase

### MongoDB Atlas Setup
- [ ] Account created at mongodb.com/cloud/atlas
- [ ] M0 Sandbox cluster created
- [ ] Database user created (username: `admin`)
- [ ] Connection string copied
- [ ] IP whitelist set to `0.0.0.0/0` (Allow from anywhere)

### Google Cloud OAuth Setup
- [ ] Project created in Google Cloud Console
- [ ] OAuth 2.0 credentials created
- [ ] `CLIENT_ID` saved
- [ ] `CLIENT_SECRET` saved
- [ ] Test user added to OAuth app (if in development)

### GitHub Repository Setup
- [ ] GitHub account created
- [ ] Repository created
- [ ] Code pushed to GitHub
- [ ] Repository is public (for ease of linking to Render/Railway)

  - [ ] Connection string verified

## 🆘 Troubleshooting Checklist

### Build Fails
- [ ] Check Render logs for error messages
- [ ] Verify `package.json` is valid JSON
- [ ] Ensure all dependencies are in `package.json`
- [ ] Try local build: `npm run build`

### App Crashes After Deploy
- [ ] Check Render logs for error details
- [ ] Verify environment variables are set correctly
- [ ] Test MongoDB connection string locally
- [ ] Check if `dist/` folder is properly built

### Cannot Connect to Database
- [ ] Verify MongoDB URI in Render environment variables
- [ ] Check IP whitelist in MongoDB Atlas is set to `0.0.0.0/0`
- [ ] Verify MongoDB cluster is running
- [ ] Test connection locally with same URI

### Google OAuth Not Working
- [ ] Verify redirect URIs are added in Google Cloud Console
- [ ] Check `CLIENT_ID` and `CLIENT_SECRET` match
- [ ] Ensure they're set in Render environment variables
- [ ] Check if user is in OAuth test list

### App Too Slow or Sleeping
- [ ] Normal on Render free tier (sleeps after 15 mins)
- [ ] First request takes 30+ seconds to wake up
- [ ] Upgrade to paid tier to prevent sleeping ($7/month)
